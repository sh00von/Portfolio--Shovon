Predicting Rainfall Patterns Using LSTM Networks: A Deep Dive into Our Sylhet Case Study

How we combined hydrological domain expertise with deep learning to build a real-time rainfall prediction system for the Sylhet and Sreemangal region, and what the results tell us about climate variability.

Background & Motivation

Bangladesh is one of the most climate-vulnerable countries in the world. Unpredictable monsoon patterns, flash floods, and changing rainfall distributions continue to pose significant challenges for agriculture, infrastructure, and human settlements. The Sylhet and Sreemangal regions are particularly affected — receiving some of the highest annual rainfall in the country.

Our research started with a simple question: can we build a system that reliably predicts short-term and medium-term rainfall patterns using historically available meteorological data? Traditional statistical methods (ARIMA, exponential smoothing) fall short when rainfall exhibits non-linear behaviors. This led us to explore deep learning — specifically Long Short-Term Memory (LSTM) networks.

"The question is not whether climate will change — it already has. The question is whether our tools can keep pace."

The Dataset

We worked with 40+ years of daily rainfall data from the Bangladesh Meteorological Department (BMD), specifically from two stations: Sylhet (1980–2023) and Sreemangal (1980–2023). After cleaning and preprocessing, our dataset contained:

  • Features: daily precipitation (mm), temperature, humidity, wind speed
  • Target variable: 7-day and 30-day cumulative rainfall
  • Total samples: ~31,000+ daily records per station
  • Missing data rate: 3.2% (imputed using temporal interpolation)
Data Note

The BMD data required significant preprocessing. Station-level inconsistencies (equipment changes, recording gaps during political events) meant we had to build a custom validation pipeline before training.

Methodology

Our pipeline followed a four-stage approach:

  1. Data preprocessing — cleaning, normalization (MinMaxScaler), and windowed sequence generation
  2. Feature engineering — lag features, rolling means, seasonal decomposition
  3. Model training — LSTM with attention, trained on 80/10/10 splits
  4. Evaluation — RMSE, MAE, and R² as primary metrics

For sequence generation, we used a lookback_window of 60 days — meaning the model sees the past 60 days of data to predict the next 7 or 30 days.

The architecture uses a stacked LSTM with an attention layer. Here's the simplified structure:

Hyperparameter Choices

We experimented with several configurations before settling on the final architecture:

Parameter Value Notes
LSTM layers 2 (stacked) 3 layers showed overfitting
Hidden units 128 → 64 Diminishing returns above 128
Dropout 0.2 0.3 hurt convergence speed
Lookback window 60 days Best balance for seasonal capture
Learning rate 1e-3 → 1e-5 Cosine annealing scheduler
Batch size 32 GPU memory constrained

Results & Discussion

The model achieved strong performance on both stations, with the 7-day forecast outperforming the 30-day horizon as expected:

Metric Sylhet (7-day) Sylhet (30-day) Sreemangal (7-day)
RMSE 12.4 mm 38.7 mm 10.8 mm
MAE 8.1 mm 24.3 mm 7.2 mm
R² 0.91 0.78 0.93

The R² of 0.91 for 7-day Sylhet predictions is particularly encouraging. The model captures pre-monsoon build-up patterns well, though it struggles with extreme outlier events (e.g., the 2022 flash flood anomaly in Sylhet).

Interestingly, the Sreemangal station data produced better results despite having more missing values. We attribute this to Sreemangal's more consistent seasonal cycle compared to Sylhet's erratic urban heat island effects.

Lessons Learned

A few key takeaways from this research:

  • Domain knowledge matters more than architecture. The biggest accuracy gains came from better feature engineering (seasonal decomposition, humidity-rainfall lag correlations), not from making the model larger.
  • Data quality is non-negotiable. We spent 60% of our project time on data cleaning. BMD station data has legitimate gaps and instrument changes that naive imputation can't handle.
  • Simpler models can surprise you. A well-tuned GRU model with the same features achieved R² = 0.88 — only slightly behind our LSTM. The marginal gain from LSTM wasn't always worth the training cost.
  • Evaluation on extreme events is critical. Standard metrics like RMSE paint an optimistic picture. We added a custom "extreme event capture rate" metric and saw our model only caught 62% of anomalous rainfall events.

What's Next

We're currently working on three extensions of this research:

  1. Multi-station fusion — combining data from 10+ BMD stations using a graph neural network to capture spatial dependencies
  2. Satellite data integration — incorporating CHIRPS and GPM satellite precipitation data for regions without ground stations
  3. Real-time dashboard — a Next.js + Python FastAPI app that serves live predictions with uncertainty intervals
